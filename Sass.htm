Sass和SCSS区别
1、后缀名不同，很好理解
2、Sass以严格的缩进语法规则书写，不带大括号和分号；而SCSS的语法规则和CSS的语法很类似。
Sass:
  1. $font-stack: Helvetica, sans-serif  //定义变量
  2. $primary-color: #333  //定义变量
  4. body
  5.   font: 100% $font-stack
  6.   color: $primary-color
SCSS:
  1. $font-stack: Helvetica, sans-serif;
  2. $primary-color: #333;
  4. body {
  5.   font: 100% $font-stack;
  6.   color: $primary-color;
  7. }
编译出来的CSS：
  1. body {
  2.   font: 100% Helvetica, sans-serif;
  3.   color: #333;
  4. }
SCSS写法和CSS没什么区别，所以Sass大家更喜欢（什么鬼？）。简单说就是，你把.css改成.scss也能用。

声明变量
三个部分：1、声明变量的符号"$"2、变量名称3、赋予变量的值

  1. $brand-primary : darken(#428bca, 6.5%) !default; // #337ab7
  2. $btn-primary-color : #fff !default;
  3. $btn-primary-bg : $brand-primary !default;
  4. $btn-primary-border : darken($btn-primary-bg, 5%) !default;
在后面加上 !default表示默认变量
覆盖默认值：在默认变量之前重新声明变量即可
  1. $baseLineHeight: 2;
  2. $baseLineHeight: 1.5 !default;
  3. body{
  4.     line-height: $baseLineHeight; 
  5. }
编译后的CSS：
  1. body{
  2.     line-height:2;
  3. }
调用变量："$变量名"放在你想调用的地方就行了。
局部变量和全部变量
没什么难理解的，看一下代码就行了
  1. $color: orange !default;//定义全局变量(在选择器、函数、混合宏...的外面定义的变量为全局变量)
  2. .block {
  3.   color: $color;//调用全局变量
  4. }
  5. em {
  6.   $color: red;//定义局部变量
  7.   a {
  8.     color: $color;//调用局部变量
  9.   }
  10. }
  11. span {
  12.   color: $color;//调用全局变量
  13. }
全局变量的影子：简单说就是局部变量的名字和全局变量的名字一样。
什么时候声明变量？
    1、该值至少重复出现了两次；
    2、该值至少可能会被更新一次；
    3、该值所有的变现都与变量有关（非巧合）
嵌套
选择器嵌套
有这样一段结构
  1. <header>
  2. <nav>
  3.     <a href=“##”>Home</a>
  4.     <a href=“##”>About</a>
  5.     <a href=“##”>Blog</a>
  6. </nav>
  7. <header>
想选中header中的a标签，CSS
  1. nav a {
  2.   color:red;
  3. }
  5. header nav a {
  6.   color:green;
  7. }
Sass：
  1. nav{
  2.   a{
  3.     color: red;
  5.     header & {
  6.       color: green;
  7.     }
  8.   }
  9. }
属性嵌套
CSS有些属性只是后缀不一样，前缀一样，比如margin-top/margin-bottom
如果CSS是这样：
  1. .box {
  2.     border-top: 1px solid red;
  3.     border-bottom: 1px solid green;
  4. }
那么Sass可以这样写：
  1. .box{
  2.   border: {
  3.     top: 1px solid red;
  4.     bottom: 1px solid green;
  5.   }
  6. }
伪类嵌套
和属性嵌套差不多，加上一个&符号就可以了
  1. .clearfix{
  2. &:before,
  3. &:after {
  4.     content:"";
  5.     display: table;
  6.   }
  7. &:after {
  8.     clear:both;
  9.     overflow: hidden;
  10.   }
  11. }
避免选择器嵌套：俩原因，读不懂，不会写。

混合宏
需要重复大量的样式时，混合宏很有用
声明混合宏
不带参数混合宏
使用"@mixin"来声明
  1. @mixin border-radius{
  2.     -webkit-border-radius: 5px;
  3.     border-radius: 5px;
  4. }
带参数混合宏
  1. @mixin border-radius($radius:5px){
  2.     -webkit-border-radius: $radius;
  3.     border-radius: $radius;
  4. }
复杂的混合宏
可以在大括号里写上带有逻辑关系的语句
  1. @mixin box-shadow($shadow...){
  2.   @if length($shadow) >= 1 {
  3.     @include prefixer(box-shadow, $shadow);
  4.   } @else{
  5.     $shaow: 0 0 4px rgba(0,0,0,0.3);
  6.     @include prefixer(box-shadow, $shadow);
  7.   }
  8. }
box-shadow带有多个参数，可以用"..."来代替。当$shadow的参数数值量大于或等于1时，表示有多个阴影值，反之调用默认的参数值0 0 4px rgba(0,0,0,0.3)。
调用混合宏
匹配一个关键词"@include"来调用
比如调用上面的混合宏border-radius：
  1. button {
  2.     @include border-radius;
  3. }
混合宏的参数
传一个不带值的参数
  1. @mixin border-radius($radius){
  2.   -webkit-border-radius: $radius;
  3.   border-radius: $radius;
  4. }
调用时给混合宏传一个参数值：
  1. .box{
  2.   @include border-radius(3px);
  3. }
传一个带值的参数
  1. @mixin border-radius($radius:3px){
  2.   -webkit-border-radius: $radius;
  3.   border-radius: $radius;
  4. }
调用时可以直接调用混合宏"border-radius"
  1. .btn{
  2.   @include border-radius;
  3. }
当然也可以在调用时再给混合宏的参数传值：
  1. .box{
  2.   @include border-radius(50%);
  3. }
传多个参数
  1. @mixin center($width,$height){
  2.   width: $width;
  3.   height: $height;
  4.   position: absolute;
  5.   top: 50%;
  6.   left: 50%;
  7.   margin-top: -($height) / 2;
  8.   margin-left: -($width) / 2;
  9. }
一个特别的参数"..."，当混合宏传入的参数过多时，可以用这个参数来代替。
  1. @mixin box-shadow($shadow...){
  2.   @if length($shadows) >= 1 {
  3.     -webkit-box-shadow: $shadows;
  4.     box-shadow: $shadows;
  5.   } @else {
  6.     $shadows: 0 0 2px rgba(#000,.25);
  7.     -webkit-box-shadow: $shadow;
  8.     box-shadow: $shadow;
  9.   }
  10. }
混合宏的不足
生成冗余的代码块
  1. @mixin border-radius{
  2.   -webkit-border-radius: 3px;
  3.   border-radius: 3px;
  4. }
  6. .box {
  7.   @include border-radius;
  8.   margin-bottom: 5px;
  9. }
  11. .btn {
  12.   @include border-radius;
  13. }
生成的CSS
  1. .box {
  2.   -webkit-border-radius: 3px;
  3.   border-radius: 3px;
  4.   margin-bottom: 5px;
  5. }
  7. .btn {
  8.   -webkit-border-radius: 3px;
  9.   border-radius: 3px;
  10. }

扩展/继承
加一个"@extend"就可以了
  1. .btn {
  2.   border: 1px solid #ccc;
  3.   padding: 6px 10px;
  4.   font-size: 14px;
  5. }
  7. .btn-primary {
  8.   background-color: #f36;
  9.   color: #fff;
  10.   @extend .btn;
  11. }
  13. .btn-second {
  14.   background-color: orange;
  15.   color: #fff;
  16.   @extend .btn;
  17. }
编译后的CSS
  1. .btn, .btn-primary, .btn-second {
  2.   border: 1px solid #ccc;
  3.   padding: 6px 10px;
  4.   font-size: 14px;
  5. }
  7. .btn-primary {
  8.   background-color: #f36;
  9.   color: #fff;
  10. }
  12. .btn-second {
  13.   background-clor: orange;
  14.   color: #fff;
  15. }
占位符 %placeholder
%声明的代码如果不被@extend调用，不会产生任何的代码。
  1. %mt5 {
  2.   margin-top: 5px;
  3. }
  4. %pt5{
  5.   padding-top: 5px;
  6. }
  8. .btn {
  9.   @extend %mt5;
  10.   @extend %pt5;
  11. }
  13. .block {
  14.   @extend %mt5;
  16.   span {
  17.     @extend %pt5;
  18.   }
  19. }
编译后的CSS：
  1. .btn, .block {
  2.   margin-top: 5px;
  3. }
  5. .btn, .block span {
  6.   padding-top: 5px;
  7. }
通过@extend调用的占位符，编译出的代码会将相同的代码合并在一起。

  2. @mixin set-value($side, $value) {
  3.     @each $prop in $properties {
  4.         #{$prop}-#{$side}: $value;
  5.     }
  6. }
  7. .login-box {
  8.     @include set-value(top, 14px);
  9. }
编译成CSS：
  1. .login-box {
  2.     margin-top: 14px;
  3.     padding-top: 14px;
  4. }
